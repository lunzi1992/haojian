# 👶 宝宝出行助手 V2 产品需求文档（中国大陆 · 0–1岁）

---

## 一、版本目标

### 1.1 核心目标

在现有 MVP 基础上完成三件事：

1. **提升判断准确性**（引入评分系统 + 湿度 + 体感温度）
2. **提升用户信任感**（解释原因 + 人话总结）
3. **提升使用频率**（增加最佳外出时间推荐）

---

## 二、产品定位

### 2.1 目标用户
- 中国大陆地区家长
- 0–1岁婴儿（重点关注：0–6个月）

### 2.2 使用边界
- 本产品仅针对婴幼儿环境建议
- 对1岁以上儿童的建议可能偏保守

---

## 三、功能需求

### 3.1 年龄体系

#### 功能描述
根据宝宝出生日期自动计算月龄，并划分阶段。

#### 年龄分层

| 月龄     | 阶段   | 特征     |
| -------- | ------ | -------- |
| 0–3个月  | 新生儿 | 高敏感   |
| 3–6个月  | 小婴儿 | 中高敏感 |
| 6–12个月 | 婴儿   | 中等敏感 |

#### 展示示例
**👶 4个月 · 小婴儿阶段（对环境较敏感）**

---

### 3.2 风险评分系统（核心）

#### 功能描述
基于环境数据计算风险分数，并输出建议等级。

#### 输入参数

| 参数                   | 是否必需 |
| ---------------------- | -------- |
| 温度（temp）           | 是       |
| 体感温度（feels_like） | 是       |
| 紫外线（uv）           | 是       |
| 空气质量（aqi）        | 是       |
| 风速（wind_speed）     | 是       |
| 湿度（humidity）       | 是       |

#### 评分规则（V2）

```python
score = 0

# 温度
if temp > 32:
    score += 25
if temp < 5:
    score += 30

# 体感温度
if feels_like > 35:
    score += 20

# 紫外线
if uv >= 7:
    score += 25
elif uv >= 4:
    score += 10

# 空气质量
if aqi > 100:
    score += 30
elif aqi > 50:
    score += 15

# 风速
if wind_speed > 25:
    score += 10

# 湿度
if humidity > 80:
    score += 10
if humidity > 90:
    score += 15
```

#### 年龄权重

```python
if age <= 3:      # 个月
    score *= 1.5
elif age <= 6:
    score *= 1.2
else:
    score *= 1.0
```

#### 输出分级

| 分数  | 建议         |
| ----- | ------------ |
| 0–30  | ✅ 适合外出   |
| 30–60 | ⚠️ 谨慎外出   |
| >60   | ❌ 不建议外出 |

### 3.3 原因解释模块

#### 功能描述
展示影响结果的关键因素。

#### 示例

```
主要原因：
• 紫外线指数 9（过高，容易晒伤）
• 空气质量 AQI 120（对呼吸道有影响）
• 湿度较高（闷热不适）
```

---

### 3.4 行动建议模块

#### 功能描述
提供可执行的外出建议。

#### 示例

```
建议：
• 尽量选择阴凉区域
• 控制外出时间在20分钟以内
• 使用遮阳车篷或遮阳伞
```

---

### 3.5 今日最佳外出时间（新增）

#### 功能描述
推荐更适合外出的时间段。

#### 示例

```
今日较适合外出时间：
17:30 - 19:00

原因：
• 紫外线降低
• 温度更舒适
```

#### 实现逻辑
1. 获取未来6小时天气数据
2. 每小时计算评分
3. 选择评分最低的连续时间段

---

### 3.6 人话总结模块

#### 功能描述
用一句话总结当前环境情况（通俗易懂）。

#### 示例

```
今天天气较闷热且紫外线较强，
不太适合带4个月宝宝外出。
```

---

### 3.7 数据获取与优化

#### 必须字段

| 字段       | 用途     |
| ---------- | -------- |
| feels_like | 体感温度 |
| humidity   | 湿度     |

#### 数据策略
- 天气数据与空气质量数据分开获取
- 缓存时间：15分钟
- 支持手动刷新

## 四、非功能需求

### 4.1 性能
- 页面加载时间 < 2秒
- 数据请求时间 < 1秒

### 4.2 容错处理
- API失败时需友好提示
- 显示“暂无数据”状态

### 4.3 合规要求
- 必须包含免责声明（如“本建议仅供参考，不能替代专业医疗意见”）

---

## 五、开发优先级

### P0（必须）
- 风险评分系统（含湿度 + 年龄权重）
- 原因解释模块
- 首页结构优化

### P1（重要）
- 最佳外出时间推荐
- 人话总结模块

### P2（优化）
- UI美化
- 动画效果

---

## 六、成功指标

- 用户是否理解建议（页面停留时长）
- 用户是否每天打开（留存率）
- 用户主观信任度（反馈或评分）

---

## 七、版本总结

V2的目标不是增加复杂度，而是显著提升用户体验：

- 从“能用” → **“好用”**
- 从“数据展示” → **“决策辅助”**
- 从“工具” → **“安心感产品”**
